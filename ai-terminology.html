Hvad betyder termerne, hvordan spiller de sammen, og hvad er de typiske misforståelser?

AI
01 / 12
AI
Artificial Intelligence
Hvad er det?

Kunstig intelligens er det brede forskningsfelt der handler om at få maskiner til at udføre opgaver, der normalt kræver menneskelig intelligens — genkendelse, beslutningstagning, sprogforståelse, planlægning.

AI er paraplyen. Under den finder du maskinlæring, deep learning, LLM'er, computervision, robotteknologi og meget mere. Når folk siger "AI" i dag, mener de oftest LLM-baserede produkter — men feltet er langt bredere.

⚠ Typisk forvirring

"AI kan tænke og føle"

AI processerer mønstre i data — den tænker eller føler ikke. Den er et avanceret værktøj der kan simulere forståelse, men den har ingen bevidsthed, intentioner eller ønsker.

LLM
02 / 12
LLM
Large Language Model
Hvad er det?

En stor sprogmodel er en type AI-model der er trænet på enorme mængder tekst for at forstå og generere sprog. Den forudsiger det mest sandsynlige næste ord baseret på konteksten.

LLM'er som GPT-4, Claude og Llama er "motoren" bag moderne AI-assistenter. De kan skrive, analysere, oversætte og ræsonnere — men de er stadig bare ét værktøj i AI-værktøjskassen.

✕ Typisk misforståelse

"LLM = AI"

LLM er én type AI — ligesom en benzinmotor er én type motor. AI er paraplyen der dækker alt fra billedgenkendelse til robotstyring. En LLM er specialiseret i sprog.

Token
03 / 12
Token
Tokens / Context Window
Hvad er det?

Den mindste enhed en LLM arbejder med. Ikke et ord, ikke et bogstav — noget midt imellem. En token er ca. ¾ af et engelsk ord. Danske ord med æ, ø, å bruger ofte flere tokens.

En LLM har et "context window" — det maksimale antal tokens den kan behandle ad gangen, både input og output. Alt hvad agenten læser og skriver i én samtale skal passe inden for dette vindue.

⚠ Typisk forvirring

"Token = ord"

Ordet "Marmalade" kan være 3 tokens. "Kontraktgennemgang" kan være 4-5. Tokens påvirker både pris og hvor meget information en agent kan behandle ad gangen.

"Fruity AI" → 3 tokens
"konkurrentanalyse" → 5 tokens
Prompt
04 / 12
Prompt
System Prompt / User Prompt
System Prompt

Den skjulte instruktion der definerer agentens rolle, regler og kontekst. Brugeren ser den normalt ikke.

System prompten er som en jobbeskrivelse: den fortæller agenten hvem den er, hvad den må, og hvordan den skal opføre sig. Den sætter rammerne for alle svar.

Eksempel: "Du er en juridisk assistent for et dansk advokatfirma. Svar altid på dansk. Henvis aldrig til lovgivning du ikke er sikker på."

User Prompt

Det brugeren faktisk skriver — spørgsmålet, opgaven eller instruktionen der sendes til agenten i samtalen.

User prompten er det synlige input. Jo mere kontekst, eksempler og struktur brugeren giver, desto bedre bliver svaret. En god prompt er mere end bare et spørgsmål — det er en detaljeret opgavebeskrivelse.

Eksempel: "Gennemgå denne kontrakt og list de 5 vigtigste risici for os som leverandør. Formatér som tabel."

✕ Typisk misforståelse: "Prompt = bare et spørgsmål" — En prompt er instruktion + kontekst + eksempler + regler. Kvaliteten af prompten afgør kvaliteten af svaret.
Hallucination
05 / 12
Hallucination
Konfabulation / Fabrication
Hvad er det?

Når en LLM genererer tekst der lyder overbevisende og selvsikker, men som er faktuel forkert. Modellen "ved" ikke den tager fejl — den producerer de mest sandsynlige tokens uden et begreb om sandhed.

Hallucinationer er en iboende egenskab ved LLM'er, ikke en bug der kan fjernes helt. Det er derfor RAG, kildeverifikation og menneskelig oversight er afgørende i professionelle sammenhænge.

✕ Typisk misforståelse

"AI lyver med vilje"

AI har ingen intention om at bedrage. Hallucinationer opstår fordi modellen genererer sandsynlig tekst, ikke verificerede fakta. Den kan opfinde kilder, tal og referencer der aldrig har eksisteret.

DEL 02

Agenten i praksis

Hvordan en LLM bliver til en selvstændig medarbejder der kan handle, hente viden og løse opgaver.

Agent
06 / 12
Agent
Autonomous AI Agent
Hvad er det?

En AI-agent er software der kan planlægge, ræsonnere og handle selvstændigt for at løse komplekse opgaver. Den bruger en LLM som hjerne, men kan også kalde tools, hente data og tilpasse sin tilgang undervejs.

Forskellen på en chatbot og en agent: en chatbot svarer på spørgsmål. En agent tager ansvar for en opgave, lægger en plan, udfører handlinger og vurderer resultatet — som en dygtig medarbejder der arbejder selvstændigt.

✕ Typisk misforståelse

"En chatbot er en agent"

En chatbot reagerer passivt på input. En agent handler proaktivt — den planlægger, bruger tools, tilgår data og fuldfører opgaver fra start til slut uden konstant vejledning.

RAG
07 / 12
RAG
Retrieval-Augmented Generation
Hvad er det?

En teknik der giver en LLM adgang til ekstern viden ved at søge i dokumenter, databaser eller filer — og indsætte de relevante uddrag direkte i prompten.

I stedet for at stole på træningsdata alene, "slår agenten op" i jeres faktiske data før den svarer. RAG reducerer hallucinationer markant fordi svaret bygger på reel information.

✕ Typisk misforståelse

"RAG = bare en søgning"

RAG er mere end søgning. Den finder relevante tekstuddrag, rangerer dem, og væver dem ind i prompten så LLM'en kan svare med kontekst den aldrig er trænet på — f.eks. jeres interne kontrakter eller projektdata.

DEL 03

Værktøjer & forbindelser

Hvad en agent kan gøre, og hvordan den forbinder til omverdenen.

Tool
08 / 12
Tool
Function / External Action
Hvad er det?

Et tool er en specifik handling en agent kan udføre — som at søge på nettet, læse en fil, sende en e-mail eller slå op i en database. Tools udvider agentens evner ud over ren tekstgenerering.

Uden tools kan en LLM kun skrive tekst. Med tools kan en agent faktisk gøre ting i den virkelige verden — hente data, opdatere systemer, sende beskeder og meget mere.

web_search
Søger på internettet og returnerer relevante resultater.
read_file
Læser indholdet af en fil fra et tilknyttet system.
send_email
Sender en e-mail via den tilknyttede mailkonto.
Skills
09 / 12
Skills
Skill = Tool + Viden + Regler
Hvad er det?

En skill er en pakke der kombinerer tools med domæneviden og regler. Hvor et tool er en enkelt handling (f.eks. "søg i database"), er en skill en hel kompetence (f.eks. "lav konkurrentanalyse").

Skills gør en agent specialiseret. De indeholder viden om hvornår og hvordan tools skal bruges i en bestemt kontekst — ligesom en medarbejder der både har værktøjer og ekspertisen til at bruge dem rigtigt.

✓ Eksempler på skills
  • → Konkurrentanalyse fra offentlige kilder
  • → Kontraktgennemgang med risiko-scoring
  • → Ugentlig projektrapport fra Jira + Slack
  • → Kundeonboarding med tjeklistevalidering
  • → SEO-audit med handlingsanbefalinger
MCP
10 / 12
MCP
Model Context Protocol
Hvad er det?

MCP er en åben standard der definerer hvordan AI-agenter forbinder til eksterne tools og datakilder. Tænk på det som USB for AI — en universel stikforbindelse.

Før MCP skulle hver integration bygges fra bunden. Med MCP kan en agent tilslutte sig ethvert kompatibelt system via en standardiseret protokol — uden specialkode for hvert enkelt tool.

⚠ Typisk forvirring

"MCP = en API"

En API er forbindelsen til ét system. MCP er protokollen der standardiserer forbindelsen til mange API'er. MCP gør det muligt for agenter at tilgå hundredvis af tools via ét fælles sprog i stedet for at bygge individuelle integrationer.

DEL 04

Adgangsveje

Hvordan brugere og systemer tilgår AI-agenten — de to grænseflader.

API
11 / 12
API
Application Programming Interface
Hvad er det?

En API er et sæt regler der lader forskellige programmer tale sammen. Tænk på det som en menu på en restaurant — du behøver ikke vide hvordan køkkenet fungerer, du bestiller bare fra menuen.

API'en definerer hvilke "retter" (funktioner) der er tilgængelige, hvad du skal sende med (input), og hvad du får tilbage (output). Det er den standardiserede måde software udveksler data på.

✕ Typisk misforståelse

"API er kun for udviklere"

Du bruger API'er hver dag uden at vide det. Når du logger ind med Google, betaler med MobilePay eller tjekker vejret i en app — så taler apps sammen via API'er bag kulisserne.

CLI
12 / 12
CLI
Command Line Interface
Hvad er det?

En tekstbaseret grænseflade hvor du skriver kommandoer i stedet for at klikke. CLI giver direkte kontrol over programmer og er fundamentet for automatisering.

Mens en grafisk brugerflade (GUI) er designet til at være visuel, er CLI designet til at være effektiv. Du kan kæde kommandoer sammen, skrive scripts og automatisere gentagne opgaver med få linjer.

✕ Typisk misforståelse

"CLI er gammeldags" — CLI er hurtigere og nemmere at automatisere end GUI.
